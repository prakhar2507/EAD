Exploratory Data Analysis (EDA)

EDA Checklist

âœ…Ultimate EDA Checklist
â–¶

ðŸ”¹ 1. Context & Objective

  • Understand the business/domain problem
  • Identify target variable (if supervised task)
  • Determine type of ML problem (classification, regression, clustering, etc.)
â–¶

ðŸ”¹ 2. Data Loading & Setup

  • Load data correctly (pd.read_csv(), etc.)
  • Check shape (df.shape)
  • Preview data (df.head(), df.sample())
  • Check column names and duplicates
  • Save a copy of raw data for backup
â–¶

ðŸ”¹ 3. Data Types & Structure

  • Check df.dtypes
  • Identify categorical, numerical, datetime, and text features
  • Convert types as needed (to_datetime(), astype())
  • Detect and remove ID-like or irrelevant columns
â–¶

ðŸ”¹ 4. Missing Values

  • Use df.isnull().sum() and % missing
  • Visualize missing values (sns.heatmap, missingno.matrix)
  • Determine strategy:
  • Drop rows/columns (if valid)
  • Impute (mean/median/mode/forward fill)
  • Advanced imputation (KNN, MICE, model-based)
â–¶

ðŸ”¹ 5. Univariate Analysis

Numerical Features

  • Histograms
  • Boxplots
  • Summary stats (df.describe())

Categorical Features

  • Bar plots / Count plots
  • value_counts() for cardinality & imbalance

Target Variable

  • Distribution (histogram / countplot)
  • Class balance (classification)
  • Range and skewness (regression)
â–¶

ðŸ”¹ 6. Bivariate/Multivariate Analysis

  • Correlation matrix (numerical-numerical)
  • Scatter plots / pairplots
  • Boxplots for category vs. numeric
  • Crosstabs or grouped bar charts (cat-cat)
  • Group-based summary stats (groupby())
â–¶

ðŸ”¹ 7. Outlier Detection

  • Boxplots for outlier visualization
  • IQR method or Z-score method
  • Decide to:
  • Cap
  • Remove
  • Leave as is
  • Use log/box-cox transformation
â–¶

ðŸ”¹ 8. Feature Engineering

  • Create derived columns (ratios, interactions)
  • Extract date parts (Year, Month, Day)
  • Create bins (e.g., age groups)
  • Group rare categories
  • Aggregate features (grouped means, etc.)
â–¶

ðŸ”¹ 9. Encoding Categorical Features

  • Label encoding (if ordinal)
  • One-hot encoding (if nominal)
  • Frequency/Count encoding
  • Target/Mean encoding (with care to avoid leakage)
â–¶

ðŸ”¹ 10. Data Scaling

  • Normalize or scale numeric features (if needed):
  • StandardScaler (mean = 0, std = 1)
  • MinMaxScaler (0 to 1)
  • RobustScaler (for outliers)
  • Only scale after encoding & splitting (on train/test separately)
â–¶

ðŸ”¹ 11. Feature Selection (Optional Pre-modeling)

  • Drop low-variance features
  • Drop highly correlated features
  • Use mutual information / ANOVA F-test
  • Use RFE / model-based methods (if planning ahead)
â–¶

ðŸ”¹ 12. Visual Reporting

  • Summarize important insights visually
  • Save figures of important plots
  • Optional: use profiling tools: pandas-profiling, SweetViz, D-Tale, Autoviz
â–¶

ðŸ”¹ 13. Documentation & Summary

  • Document key insights from EDA
  • Highlight data quality issues
  • Note feature transformations
  • Make modeling recommendations
â–¶

ðŸ”¹ 14. Export Clean Data

  • Save cleaned & transformed dataset (to_csv)
  • Optionally split into train/test before modeling
